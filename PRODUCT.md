# Product

## Register

product

## Users

主要用户是**旅游公司的 OP(操作专员)**——一个约 260 人团队里的一线同事,每天的工作就是帮**中小企业客户**搜航班、比价、出票、改退。他们是**高频专业用户**,一天处理大量订单,对航司 / 舱位 / 运价规则烂熟,愿意为效率接受更高的信息密度和领域术语。他们替别人花钱、定别人的行程,所以**准确与可信是底线,比价与速度是日常**。

次要语境:TripDesk 同时是 **Rebyte 能力的模板 demo**(见 CLAUDE.md)。评估 Rebyte 的人会透过它判断"这套 agent + VM + skill 能不能做成真正的产品",所以界面要显得**可成品化、可托付**。

## Product Purpose

TripDesk 是一个 **AI 机票预订工作台**:左侧聊天驱动 agent 跑完整在线机票流程(搜索 → 验价 → 下单 → 支付 → 售后),右侧 bench 把订票领域状态渲染成卡片。数据来自 TravelKit MCP,流程由 Rebyte 沙箱子 agent 执行。

它真正要证明的不是"又一个订票网站",而是 **Rebyte 的三件能力**(跑 agent、跑 VM 沙箱、跑私有 skill)能被快速配一套垂直 UI 就成型。成功 = OP 敢用它替客户完成真实订单,评估者看完觉得"这能上线"。

## Brand Personality

**克制 · 可信 · 编辑感。** 像一份排版讲究的行程单或对账单,而不是一个促销页。安静、专业、信息清楚;靠留白、层级和单一强调色建立信任,而不是靠颜色和动效抢注意力。一个 OP 一天看它几百次都不累。

## Anti-references

明确**不要**像这三类(出自用户,逐条作为红线):

- **深色霓虹科技风**——深蓝黑底 + 电光蓝 / 霓虹绿。这正是改版前那套,绝不回头。
- **玩具 / 卡通感**——圆滚滚大圆角、强阴影、糖果色、表情堆砌。
- **OTA 花哨拥挤风**——携程 / Expedia / 飞猪那种红黄促销色块、弹窗、角标、信息过载。

## Design Principles

1. **UI 是工具结果的镜像。** bench 渲染的是 agent 真实的 `tool_result` 领域状态,不是装饰性摘要;接口没返回的就不显示。
2. **写操作必须明确确认。** 下单 / 支付 / 取消 / 退改前一律走 ConfirmGate,把"花钱那一步"交还给人。信任来自控制权。
3. **不编造未返回的数据。** 行李额、退改规则等接口没给的,如实说"未返回",绝不脑补。诚实优先于完整。
4. **为 OP 的"比价"而生:密度即效率。** 搜索结果要好比、好扫、好决策;可以密、可以用术语,别为了好看牺牲"一屏能比几个方案"。
5. **可信胜过惊艳。** 工具应当消失在任务里;克制本身是功能,不是妥协。

## Accessibility & Inclusion

- 目标 **WCAG AA**:正文对比度 ≥ 4.5:1,大字 / 非关键 ≥ 3:1(这也是 `--muted` 选 olive 而非更浅的 stone 的原因)。
- 默认**简体中文**;字体栈保留 PingFang SC / Microsoft YaHei 中文回退。
- 尊重 `prefers-reduced-motion`;动效只用于状态反馈,不做编排式入场。
- 焦点态可见:2px ink-blue 描边、offset 2px。
